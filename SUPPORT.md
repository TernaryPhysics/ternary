# Getting Support

We want contributors and operators to succeed with TernaryPhysics. Choose the channel that best fits your question or issue.

## Quick Links

- **Bug Reports & Feature Requests** – [GitHub Issues](https://github.com/TernaryPhysics/ternary/issues). Include logs, kernel version, repro steps, and expected vs. actual behavior.
- **Discussions & Design Ideas** – GitHub Discussions (or issues labeled `discussion`). Share context so others can reason about trade-offs.
- **Security Incidents** – Follow the confidential process in [SECURITY.md](SECURITY.md). Never file exploitable bugs in public issues.
- **Docs & Website Fixes** – Submit PRs directly or open an issue referencing the affected page/section.

## Community Syncs

We host periodic open calls for status updates, roadmap previews, and Q&A. Scheduling details, agendas, and recordings are published in `docs/community/MEETINGS.md` (create the file if it does not yet exist). Suggest topics via issue or discussion.

## Escalations

When you need a maintainer’s attention (e.g., blocked release, regression affecting production), tag the appropriate GitHub team (such as `@KernelPhysics/maintainers`) or request a review on your pull request/issue. Provide the context you already gathered (links, logs, configs) to reduce back-and-forth.

## Commercial Help

Enterprise SLAs, incident response, and white-glove assistance are available under commercial terms. Reach out to `sales@ternaryphysics.com` to start that conversation.

Thank you for investing time in KernelPhysics. Your feedback keeps the project reliable for everyone.
