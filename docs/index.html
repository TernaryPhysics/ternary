TernaryPhysics

AI at the kernel boundary

Ternary neural networks running inside the Linux kernel. The decision happens at the moment the packet exists.

Install
Requires Linux 5.8+ with BTF enabled
Why

Intelligence where latency matters

Sub-microsecond inference

Ternary weights ({-1, 0, +1}) mean no multiplication. Just addition and subtraction at wire speed.

Learns from outcomes

Observes what actually happens. TCP connections succeed or fail. The model learns from ground truth, not labels.

No GPU required

Runs entirely on CPU. Trains locally on your hardware. Your data never leaves your network.

Autonomous improvement

Deploy once. The feedback loop runs continuously. The model gets better every day without intervention.

Shadow mode by default

New models observe for two weeks before acting. Automatic rollback if performance degrades.

Zero config

One command to install. Auto-detects network interface. Starts observing immediately.

How it works

The closed feedback loop

1

Observe

XDP hook sees every packet. Extracts TCP signals: RTT, retransmits, window size, timing patterns. Zero-copy to userspace via BPF ringbuf.

2

Label

Tracks connection outcomes. TCP established, reset, timeout. Back-propagates labels to the signals that predicted them.

3

Train

Continuous training on labeled samples. Ternary quantization via straight-through estimators. New model ready in minutes.

4

Deploy

Hot-swap weights into BPF maps. No restart, no packet loss. Shadow validation ensures no regressions.
